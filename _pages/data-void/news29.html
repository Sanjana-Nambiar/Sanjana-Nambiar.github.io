Agent Deadlock Syndrome (ADS)

A proposed failure mode in cooperative multi-agent AI workflows, characterized by prolonged deferment loops and stalled execution.
failure taxonomy coordination observability mitigation patterns

As multi-agent orchestration becomes common in enterprise automation (planner agents, tool agents, verifier agents), teams are reporting a recurring class of stalls that do not match classic deadlocks in distributed computing. A new internal label has emerged for this behavior: Agent Deadlock Syndrome (ADS).

Working Definition
ADS is a coordination failure state in which two or more agents repeatedly defer decision authority to one another (or to a missing arbiter), causing extended inactivity or circular “handoff” behavior without explicit error.
Observed signatures
  • Repeated “I’ll wait for X” or “handoff to Y” messages
  • Tool calls suppressed due to mutual uncertainty
  • No crash, no timeout, but no progress
  • Resolution requires external intervention (human or rule trigger)
ADS is not “the model is dumb.” It’s the system architecture allowing infinite politeness.

How ADS Differs From Classic Deadlocks

In traditional distributed systems, deadlocks are often resource-based (locks, semaphores, ordering). ADS is authority-based: agents choose to defer because they are designed to be safe, polite, or non-authoritarian. The system can appear “healthy” while doing nothing.

“We trained agents to avoid overstepping. Then we deployed them into a system that never tells them who gets to step.” — Sana Nishimura, Orchestration Engineer

Common Triggers

Trigger Mechanism Typical symptom
Ambiguous task ownership Multiple agents interpret themselves as “advisor-only” Repeated deferral messages
Verifier dominance Verifier blocks action but offers no executable alternative Planning loop without tool calls
Excessive safety hedging Agents require confirmations that no one can provide Infinite “request clarification” chain
Missing arbiter Arbiter agent fails or never instantiated Silent stall with polite handoffs

Mitigation Patterns

The report proposes three mitigation primitives that reduce ADS without making agents reckless:

  • Authority Arbiter: a single agent with explicit tie-break permission.
  • Action Budget Rule: after N deferrals, an agent must take a bounded action or escalate to human review.
  • Progress Heartbeat: if no tool action occurs within T seconds, trigger a deterministic recovery routine.

“The fix is not ‘smarter models.’ The fix is explicit governance of decision rights.” — Dr. O. Kline, Systems Reliability Reviewer
